What are the different ways to select an element in the DOM?

There are different ways to select an element in the DOM. We can use getElementById and using an unique ID, we can select an element. getElementsbyClassName can be another option to select elements which are consisting under same class. Item can be selected using tag names. It will select the element by it's tag name from entire HTML file. There is also querySelector and querySelectorALL by which we can use classname, tagname or unique IDname to select an element.

What is the difference between innerHTML, innerText, and textContent ?

There are simple fine-lines between these three vital properties. By using innerHTML, it can read and write a whole chunk of HTML data from a selected element. For example, we selected a "div" by an unique ID. If we want to read the entire HTML tags inside that "div" tag, we need to use innerHTML. However, we can use this property to also write new HTML tags and informations using DOM. Unlike innerHTML, innerText will only show the text between a tag. If there is tag other than text, it will return "undefined". As like innerHTML, here we can write texts and outputs from differents functions as STRING value and insert into selected tag. textContent will return every specific characters and spacings from a text. It will count the spacings will also show that as output.

What is event delegation in the DOM?

Event Delegation is nothing but traverse through the child and parent nodes to select another element from a same parent. Let's assume a scenario for better understanding. There's a card, where there's a button, by which the task will be shown as completed and the task title will be shown as alert while clicking in the completed button. Here, if we need to fetch the title for each button clicked, we need to traverse through the whole card from the button until we found the title. Here, we can use event.target.parentNode property to reach to the parent and then by using "childNodes" property then we can select the title element as per button clicked. This is a simple understanding I got from event delegation.

What is event bubbling in the DOM?

To hit the entire tree from the very begining of the document to the end of a single tag is known as bubbling. By using bubbling, we can perform more than one event at a time with a single click. To do so, first the eventListner will trigger the child element known as event trigger. Then it travel through the whole document to capture the child element or tag. After this, it propagates through the all way up from the child element to the parent element. However, we can stop propagation if it's needed through the properties of stopPropagaton() and stopImmediatePropagation()
